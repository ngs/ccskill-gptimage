## 背景

ChatGPT サブスクユーザは **Codex CLI が built-in `image_gen` tool 経由で gpt-image-2 を呼べる** ことが判明(2026-04-25 検証)。`OPENAI_API_KEY` 不要、サブスク認証だけで動作。

ccskill-gptimage の主旨は「Claude Code から ChatGPT Image 2.0 を使う」であり、その transport を API キー必須から **Codex CLI 経由に拡張** することで、サブスク勢にもスキルが届く。

なお Codex 公式 `imagegen` skill (`~/.codex/skills/.system/imagegen/`) が既に存在するが、これは Codex ユーザ向け。本スキルは Claude Code ユーザがターゲットなので機能被りは問題ではない(別経路の競合ではなく、別ターゲットの提供)。

## 方針(共通認識)

- **完全置換ではなく 2 経路併存**(Codex / API)
- Backend 優先順位: `auto` = Codex 優先 → 上限/失敗時 API フォールバック
- ユーザ明示指定可能: `--backend codex | api | auto`
- フォールバック判定は `codex login status` + `OPENAI_API_KEY` 有無で動的に

| Backend | 条件 | コスト | 使われ方 |
|---|---|---|---|
| **Codex CLI 経由** | ChatGPT サブスク + Codex CLI インストール済み | サブスク枠内(追加課金なし) | 利用可能なら優先 |
| **OpenAI API 直叩き** | `OPENAI_API_KEY` あり + Org Verified | 従量課金 | サブスクなし、または Codex 上限ヒット時の **フォールバック** |

### Codex backend の設計制約(4 つ判明)

これらは `generate_image.py` の Codex ラッパー実装と SKILL.md 注意書きに必須反映:

1. **CLI 引数順序固定**: `codex exec -i <FILE>...` は variadic で後続の positional prompt まで画像として食う。**`prompt → -i ref.png` の順に必ず固定する**(または stdin 経由で prompt を渡す)
2. **保存パス指定不可**: agent に「`/tmp/...` に保存して」は通らない。画像は **常に `~/.codex/generated_images/<session-id>/ig_*.png`** に出力される。ラッパー側で session 終了後に最新ファイルを拾って指定パスへ移動するロジックが必須
3. **agent の保存行動が非決定的**: 1 回目の ping テストでは agent が自発的にコピーしてくれたが、Test A では「保存できませんでした」と返答だけで終わった。**プロンプトでの保存依頼は信頼してはいけない**(自前回収を前提に)
4. **size/quality の厳密制御は不可**: プロンプト経由で「1024x1536 portrait」は通る場合があるが、Test B では指定なしで 1254×1254 が返った。Codex の image_gen tool スキーマは agent からも完全には見えていない。**ピクセル単位の size 指定が必須なユースケース(印刷、広告枠等)では API backend にフォールバックすべき**
