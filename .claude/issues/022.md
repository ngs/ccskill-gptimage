## 概要

`ccskill-gptimage generate` には多数のオプションがある(`--help` 参照)。これらが
**本当に `--backend api` 経由で機能するのか** を網羅的に検証する。検証作業はそのまま
「このスキルで何ができるか(特にオプションを使った例)」を示す**使用例カタログ**になる。

今回は **API 経路(`--backend api`)に限定**。Codex 経路は対象外
(API で通らないものを Codex で見ても無意味なため、API での網羅確認が先決)。


## スコープ外

- Codex backend / auto backend の検証(別ラウンドで実施)
- 新機能の追加(本 issue は既存オプションの検証とカタログ化のみ)


## 方針

- 各オプションを「単に通る」で終わらせず、**成果物を視覚的に確認**し効果が出ているかまで見る
  (CLAUDE.md 品質方針)。
- 検証結果はそのまま **カタログ(コマンド + 結果画像 + 効きの観察 + ○/△/× 判定)** として残す。
- 成果物は **リポジトリ内 `catalog/`** に画像 + メタ JSON + Markdown で保存。
- `transparent` 背景 / `input-fidelity` は gpt-image-2 では検証不可のため、
  **`--model gpt-image-1.5` のサブテスト**を含めて「代替モデルでは効く」ことまで確認する。


## 検証対象オプション(全 14)

| # | オプション | 検証で見ること |
|----|-----------|--------------|
| 1 | `--size` | 出力画像の実ピクセルが指定どおりか(auto / 1024x1024 / 1024x1536 / 1536x1024 / 4K 3840x2160 / 3:1 端 1536x512) |
| 2 | `--quality` | low/medium/high/auto の描き込み・テキスト精度差を同一プロンプトで比較 |
| 3 | `--background` | auto/opaque は生成可。transparent は gpt-image-2 で事前ブロック(exit 2)、gpt-image-1.5 で透過生成可を確認 |
| 4 | `--output-format` | png/jpeg/webp で実ファイル形式が一致するか |
| 5 | `--output-compression` | jpeg/webp で値違い(10 vs 90)でファイルサイズが変わるか |
| 6 | `--output` | 指定ディレクトリが作られ保存されるか |
| 7 | `--output-name` | ファイル名 stem が反映されるか |
| 8 | `--reference` | 単一参照で edits API に切替・編集が効くか |
| 9 | `--reference`×N | 複数参照の合成が効くか |
| 10 | `--mask` | reference + mask で局所編集が効くか(マスク PNG は Pillow で生成) |
| 11 | `--input-fidelity` | gpt-image-2 では剥がされる([Info])ことを確認 + gpt-image-1.5 で high/low が通ることを確認 |
| 12 | `--moderation` | auto/low が受理されエラーにならないか(視覚差は観察困難な旨を明記) |
| 13 | `--model` | gpt-image-2(既定) と gpt-image-1.5 が指定どおり呼ばれるか |
| 14 | `--backend` | 今回は api のみ(auto/codex は別ラウンド) |

### 負のテスト(ガードの動作証明)

- 不正 `--size`: 16 非倍数 / 最大辺超過 / アスペクト比 >3:1 / 総ピクセル下限・上限外 → API 呼び出し前に日本語エラーで拒否
- `--mask` 単独(`--reference` 無し) → exit 1
- `--background transparent` + gpt-image-2 → exit 2
