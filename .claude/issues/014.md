## 背景

issue #002 で実 API による検証を行い、`gptimageguide.md` の以下 2 点を「実 API と乖離」として全ドキュメントを修正した:

1. `background: transparent` は gpt-image-2 で 400 (Transparent background is not supported for this model.)
2. `input_fidelity` は gpt-image-2 で 400 (The model 'gpt-image-2' does not support the 'input_fidelity' parameter.)

その後、ユーザが Web Claude に再確認した結果として以下 2 つの「再解釈」が共有され、これに基づきドキュメントを **再修正** した:

- **主張 A**: 「gpt-image-2 は常に自動で最大忠実度で入力画像を処理する仕様」(input_fidelity が *欠落* ではなく *常時 high 固定* というポジティブ解釈)
- **主張 B**: 「透過 PNG が必要なら `gpt-image-1.5` に `--model` で切替可能(旧モデルだが現役)」

しかし Web Claude は **同セッションで一度誤回答 → 訂正** を経たソースであり、訂正後の主張も独立検証されていない。私(Claude Code 側)は API エラーメッセージそのものは確認したが、「常時自動最大忠実度」「gpt-image-1.5 透過対応」を **一次資料(OpenAI 公式)で裏付けていない**。

このまま GitHub 公開・dogfooding・プレスに進むと、**もう一段の誤情報伝播リスク** がある。リリース前に一次資料ベースで決着させる。

## 検証スコープ

### 主張 A: input_fidelity の真の意味

確認したいこと:
- gpt-image-2 が入力画像を処理する忠実度は実際にどう決まるのか
- 「常時最大忠実度」は OpenAI 公式の表現か、Web Claude の創作か
- もし「常時最大」が誤りなら、実際の挙動は何か(中程度固定? モデル内部判断?)
- 編集時の入力画像トークン量は本当に増えるのか、ベースラインはいくらか

判定:
- 一次資料(OpenAI 公式ドキュメント・モデルページ・Cookbook・API リファレンス・公式 blog/changelog のいずれか)で **「always high fidelity / automatically processed at max fidelity」相当の明示記述** が確認できる → 現ドキュメントの表現を維持
- 確認できない → 表現をトーンダウン:「`input_fidelity` は gpt-image-2 で指定不可。実際の保持精度はモデル仕様。プロンプト内で『Preserve …』と書く方法が推奨」へ再々修正
- 誤りと確認できる → さらに再々々修正(姉妹スキル比較も再調整)

### 主張 B: gpt-image-1.5 透過フォールバック

確認したいこと:
- `gpt-image-1.5` は現時点で OpenAI API で利用可能か(deprecate されていないか)
- `background: transparent` を本当にサポートするか
- 同じ Python SDK (`client.images.generate(model="gpt-image-1.5", ...)`) で動くか
- Org Verification 要件や課金条件に gpt-image-2 との差はあるか

判定:
- 動作する → 現 SKILL の透過 3 択(rembg / 1.5 / nanobanana)を維持
- 1.5 が利用不可 or 透過非対応 → 透過の現実解は (a) rembg / (b) ccskill-nanobanana の 2 択に縮約
