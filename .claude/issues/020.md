## 背景

現状の README は手動 4 ステップ(venv 作成 / `CCSKILL_GPTIMAGE_DIR` 環境変数の設定 / プロジェクト毎の `.claude/skills` symlink 作成)を要求しており不親切。姉妹スキル ccskill-gmail は `setup`(`~/.local/bin` への symlink 配備)+ `skill install`(`~/.claude/skills` へのユーザレベル symlink 登録)で「clone したマシン上ならどのプロジェクトからでも使える」状態を実現している。

本 issue では ccskill-gmail のパターンを最小構成で導入し、以下を実現する:

```bash
git clone https://github.com/feedtailor/ccskill-gptimage.git
cd ccskill-gptimage
./install.sh
```

これだけで、人間にも AI(Claude Code)にも、どのディレクトリ/プロジェクトからでも使える状態にする。

## 設計

### 新規ファイル

```
ccskill-gptimage/
├── install.sh             # ワンショットインストーラー(repo 直下)
└── ccskill-gptimage       # 自己位置解決ディスパッチャ(repo 直下、bash)
```

### install.sh の処理内容

1. 自身の位置から repo ディレクトリを解決
2. python3 (>= 3.10) の存在チェック
3. `venv/` が無ければ作成し、`pip install -r requirements.txt`(冪等)
4. `~/.local/bin/ccskill-gptimage` → repo のディスパッチャへ symlink(PATH 配備)
   - `~/.local/bin` が PATH に無ければ追記手順を案内
5. `~/.claude/skills/ccskill-gptimage` → repo の `.claude/skills/ccskill-gptimage` へ symlink(ユーザレベルスキル登録 = 全プロジェクトから認識)
6. backend 利用可否の診断表示(Codex CLI + サブスク認証 / OPENAI_API_KEY)。どちらも無い場合は警告 + README 該当箇所を案内(失敗にはしない)
7. 完了サマリ表示

再実行は冪等(symlink は無確認で張り直し)。ただし symlink 先が本スキル以外の実体(SKILL.md を持たないディレクトリ等)の場合は壊さずエラー。

### ディスパッチャ ccskill-gptimage のサブコマンド(最小)

| サブコマンド | 内容 |
|---|---|
| `generate [args...]` | `venv/bin/python generate_image.py "$@"` を exec(venv 未構築ならエラー + `./install.sh` 案内) |
| `uninstall` | `~/.local/bin` と `~/.claude/skills` の symlink を除去(repo と venv は残す) |
| `version` | バージョン表示(VERSION ファイルが無い間は git short hash) |
| `help` | ヘルプ表示 |

ディスパッチャは symlink を readlink で辿って自身の実体位置を解決し、`CCSKILL_GPTIMAGE_DIR` を export する(環境変数の手動設定を不要化)。
