## 背景

issue #015 Phase 4 の効果検証で **19_urban_tokyo_rainy_night** を v3 にリライトした際、以下の知見を得た:

- v2 の Constraints `no readable specific text on signs (kanji shapes only)` が**強すぎた**
- 「読めない kanji 形状のみ」を強制すると、文化的アイコン要素(店舗看板の日本語感)まで失われる
- v3 で Constraints を「読める日本語、ただし架空の固有名詞」に変更したところ、**渋谷スクランブル交差点が本物のように再現**された
  - `カラオケ 747` `居酒屋 2F・3F` `薬方 くすり・化粧品` `牛繁 焼肉` `コンタクトのアイシティ` 等の自然な日本語看板群

この知見は SKILL の他のユースケース(広告、店内モックアップ、街並みシーン等)でも活きる汎用的なものなので、`prompts/` 配下に体系化したい。

## 目的

「文化圏特有の雰囲気要素(日本語看板、街並みの言語的アイコン)」を画像に取り入れる際の実用ガイドを `prompts/` に追加し、Claude が自然な"その国らしさ"を出せるようにする。

## 追加: Phase 4 後の dogfooding(マカロニ + パッケージ合成)から得た汎用知見

2026-04-24 に Trader Joe's マカロニ&チーズの皿写真にパッケージ箱を後付け合成する dogfooding を 4 ターン実施(v1〜v4)。19_v3 のリライトと同様、SKILL/prompts に**汎用的に効く知見**が得られたので #016 のスコープに追加する。

### 追記候補 1: Negative-space declaration(空状態の肯定形明示)

**背景**:
- 元写真の Thermos マグは中身空っぽだったが、v1 では gpt-image-2 が**勝手に茶色の液体を描き入れた**(モデルが「マグ = 飲み物」を補完)
- v2 以降で `mug is empty, no liquid visible inside` と明示したら正しく空のマグになった

**汎用化**:
- 「無人の部屋」「空の皿」「空のグラス」「白紙のキャンバス」など、**null 状態を肯定形で書く**のがベスト
- ❌ `do not show people in the room`
- ✅ `the room is empty, no people present, only furniture visible`

**追記先**: `prompts/photorealism.md` または `prompts/scene-composite.md` に「Empty / null state declaration」セクション。あるいは SKILL.md の Prompt Design 10 原則の中の「Constraints」節に補足例として。

### 追記候補 2: 物理スケール明示の有効性

**背景**:
- v1 ではパッケージ箱がずんぐりして実物より丸かった
- v2 で `about 8-9 inches tall, portrait aspect ratio (3:1)` と物理スケール + 比率を明示したら、実物に近い縦長になった

**汎用化**:
- 「8 inches tall」「5 cm wide」「portrait 3:1」「A4 size」「smartphone-sized」など、**物理寸法 / アスペクト比を具体的に書くと比率が安定**
- 抽象的に "tall" / "small" と言うより数値具体化が圧倒的に効く

**追記先**: SKILL.md の Prompt Design 10 原則の「Specificity + Quality Cues」節に「**physical scale / aspect ratio specification**」として追記。`prompts/character-and-concept.md`(コレクタブル玩具のスケール)で軽く触れているが、全 prompts/ に響く原則として強調すべき。

### 追記候補 3: 反復編集での「複数同時修正」アンチパターン

**背景**:
- v1 から v2 へ: 「箱を縦長に」+「スプーン長く」を**同時に**指示
- 結果: 箱とスプーンは指示通り、しかし**皿が歪み + 箸置きが崩れる**という new な副作用
- v2 から v3 へ: 「皿を円に」+「マカロニのチーズ感」を1点ずつ集中したら良化

**汎用化**:
- これは Cookbook の `Iterate with small, single-change refinements` 原則の dogfooding 実証
- **2 点以上を同時に直そうとすると、片方を直すのに引き換えに他で副作用が出る**
- 既に SKILL.md に書いてあるが、**「実体験ベースで強調する」価値あり**

**追記先**: SKILL.md の「Goal description + small iterative refinements」節に「**実証注記: マカロニ合成の v1→v2 で 2 点同時修正 → 副作用 2 件発生。1 点ずつ反復が確実**」を 1 行注記。

### 追記候補 4(任意): Multi-Image Inputs ラベルの混乱回避

**背景**:
- プロンプト内で `Image 1: ... / Image 2: ...` というラベルを使うが、これが **モデル名「ChatGPT Images 2.0 / gpt-image-2」と読み間違えやすい**
- 2026-04-24 の dogfooding 中に実際に「Image 2 を使っていない?」とユーザに混乱されかけた

**汎用化**:
- `Image 1 / Image 2` の代わりに `Reference A / Reference B` や `Photo 1 / Product 2` のような **役割名込みのラベル**にすると曖昧さが減る
- ただし Cookbook 公式は `Image 1 / Image 2` 形式を推奨しているので、**役割名を併記**するのが折衷案

**追記先**: `prompts/scene-composite.md` の Multi-Image Inputs 節に「**ラベルにモデル名と紛らわしいインデックスを使うときは役割名を併記する**」のヒントを追加。

### 追記候補 5: マスク edit の実効性に関する dogfooding 結論 ✅ 反映済み

**背景**:
- v4 の箸置きが「フォークの歯」と一貫して誤解釈されていたため、v5 で `--mask` でその領域だけ regenerate を試行
- マスク作成: Pillow で楕円透明領域を持つ PNG を生成(本実験のみ、`requirements.txt` 不変)

**結果**:
- マスク外(箱・皿・マグ・スプーン)はピクセル変化ほぼなし、Cookbook 記述「mask は guidance」が実用上は十分に保護として機能
- マスク内(箸置き)は **明確に陶器寄りに改善**(青みがかったブルーグレー、wave 形に近づき、フォーク歯ではなくなった)
- 完璧ではないが、`--reference` のみで 4 ターン破れなかったバイアスを 1 ターンで部分的に突破 → **マスクは「効くケースもある」guidance として有効**

**SKILL への反映(2026-04-25 commit 済み)**:
- SKILL.md / SKILL.ja.md の「Local edit strategy」表に「マスク edit」の行を追加
- 「Mask edit — focused regeneration of a small region」セクション新設
- マスク作成は skill 範囲外(ユーザが Pixelmator 等で作る前提)を明記
- 推奨ワークフロー(low → high 反復)を記載
- Cookbook 引用 + dogfooding 実証(箸置き)を記載

**スキル本体への機能組み込みは見送り**:
- マスク**作成**機能は skill に内包しない(依存追加 / UI 設計の重さに見合わない)
- マスク**使用**機能(`--mask` オプション)はすでに `generate_image.py` に実装済み、ノウハウだけ SKILL に内包すれば十分
