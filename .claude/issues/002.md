## 重要な発見と対応

`gptimageguide.md` の以下の記述は **実 API と乖離していた**(おそらく gpt-image-1 / 1.5 の機能を gpt-image-2 として書いていた):

1. **`background: transparent` 非対応** — gpt-image-2 は透過背景をサポートしない。回避策: (a) `gpt-image-1.5` に切替、(b) `rembg` 等で後処理、(c) ccskill-nanobanana
2. **`input_fidelity` は指定不要(常時最大忠実度で動作)** — 当初「機能欠落」と誤認したが、Web Claude 経由で OpenAI ドキュメントを再確認した結果、**gpt-image-2 は入力画像を常に自動で最大忠実度で処理する仕様** であり、パラメータ自体が不要(指定するとエラー)。これは欠落ではなく自動仕様で、構図保持の精度はむしろ強い。トレードオフは編集時の入力画像トークンが多くなる(コスト増)

### 対応(同日反映)

- `gptimageguide.md` の該当 2 セクションに「⚠️ 訂正」ブロックを追記(input_fidelity は再修正版で「常時最大忠実度」の正しい解釈に)
- `README.md / .ja.md` を実 API スペック + 「自動高忠実度」の正解釈に修正。透過の現実解 3 つ(rembg / gpt-image-1.5 / nanobanana)を案内
- `SKILL.md / .ja.md` も同様に修正。用途別パラメータ表と姉妹スキル比較表を再更新(誤誘導していた「構図保持→nanobanana」を撤回し、編集用途は ccskill-gptimage の強みとして再ポジショニング)
- `CLAUDE.md` の制約セクションを再修正
- `generate_image.py` の事前バリデーションを修正:
  - `--background transparent` で gpt-image-2 指定時はエラー停止し 3 つの選択肢を案内
  - `--input-fidelity` で gpt-image-2 指定時は **エラーではなく Info 表示で値を None に上書き**(API エラー回避)
  - `--model gpt-image-1.5` 指定時はバリデーションをスキップして 1.5 にパススルー
- 39 件のユニットテストは全て継続パス
