## 背景

#008(ccskill-gmail 型フルディスパッチャ)は #020 を踏まえてクローズしたが、その中で
**唯一、単独価値が残った機能が `doctor`** だった。診断はコスト UX(#010)とは関心が異なるため、
独立 issue として切り出す。

`ccskill-gptimage doctor` は「なぜ画像生成が動かないのか」をユーザが自分で切り分けられるように、
環境と backend の状態を一覧診断する。#020 で実装済みの dispatcher は case 文への追加だけで
`doctor` を生やせる。

## 設計方針(2026-06-11 確定 — セキュリティ整合)

セキュリティに配慮した環境では、`.env` 等の秘密ファイルの読取をツール権限で deny し、エージェントから
秘密を隔離していることがある(別ツールでの迂回もブロックされうる)。doctor はそうした環境でも安全に
機能することを設計前提とする。

→ **doctor は秘密ファイルの中身を一切読まない**。env 変数の有無とファイルの存在のみで判定する。
中身を読まない分、判定は「presence ベース(= 設定はある模様。確証は実際に `generate` した時)」
と正直に表示する。値(API キー / access_token)は**絶対に出力しない**。

→ **Org Verification はライブ判定しない**。無料で確実に状態を取得する手段は存在せず、確実な信号は
gpt-image-2 の実生成が 403 になるか(= 課金)だけ。doctor は能動チェックせず静的案内のみ出す。
当初ドラフトの `--deep`(minimal call で 403 検出)は根拠が無かったため撤回。

## 設計

`ccskill-gptimage doctor` を追加し、各項目を ✓ / ✗ / ⚠ で表示する。**いずれの項目も秘密の中身は読まない**:

| 診断項目 | 判定方法(中身を読まない) | 失敗時の案内 |
|---|---|---|
| Python venv | `$CCSKILL_GPTIMAGE_DIR/venv/bin/python` が存在し 3.10+ | `./install.sh` 再実行 |
| 依存パッケージ | venv の python で `import openai, dotenv` が通る | `./install.sh` 再実行 |
| PATH コマンド | `~/.local/bin/ccskill-gptimage` が dispatcher への symlink | `./install.sh` 再実行 |
| PATH 通過 | `~/.local/bin` が PATH に含まれる | rc ファイルに PATH 追記 |
| ユーザレベルスキル | `~/.claude/skills/ccskill-gptimage/SKILL.md` が解決可能 | `./install.sh` 再実行 |
| Codex backend | `command -v codex` あり **かつ** `~/.codex/auth.json` が**存在**(token は読まない) | `brew install codex && codex login` |
| API backend | `OPENAI_API_KEY` env 変数あり **または** repo `.env` が**存在**(中身は読まない) | `cp .env.example .env` + キー設定 |
| backend 総合 | Codex / API のうち最低 1 つが presence あり | 上記いずれかを設定 |
| Org Verification | **チェックしない**。静的案内のみ表示 | 403 が出たら Organization Settings で Verification |

### backend 行の表示文言

中身を読まないため「presence ベース」であることを明示する。例:

- `✓ API backend: likely (OPENAI_API_KEY set)` / `✓ API backend: likely (.env present; contents not inspected)`
- `✗ API backend: not configured`
- 末尾に注記: `Backend availability is presence-based. The definitive test is running an actual generation.`

### 終了コード

- backend 総合が ✗(Codex も API も presence 無し)、または venv/依存が ✗ の場合は **exit 1**
- それ以外(symlink/PATH の ⚠ 等)は **exit 0**(警告のみ)

### Org Verification の静的案内(固定文)

> If `--backend api` generation returns **403 Forbidden**, your OpenAI Organization is almost
> certainly not verified. Complete verification at Organization Settings → General → Verification.
> (doctor cannot detect this without an actual paid generation, so it is not checked here.)
